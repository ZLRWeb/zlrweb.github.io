# TechSummary 2025-07-25

## JetBrains .NET 工具的即將變革 🚀

**Source:** https://blog.jetbrains.com/dotnet/2025/07/25/upcoming-changes-to-dotnet-tools/

-   **決策驅動因素：** JetBrains 根據使用者調查、訪談及詳細使用數據，發現部分功能使用率低，而另一些功能潛力未完全發揮，因此決定精簡產品線，提升核心價值。
-   **2025.2 版本更新重點：**
    -   **dotCover 精簡化：** 獨立的 dotCover 工具將進行大幅精簡，以提升效能並降低複雜性。命令行執行器現代化，命令統一到 `dotcover cover` 介面，配置檔從 XML 轉換為純文字參數檔。
    -   **功能移除：** 移除了少用的過濾選項（方法和類別過濾、檔案路徑過濾、原始碼註解過濾），並停止支援舊版應用程式類型，包括 IIS Express、WCF、WinRT 和外部 .NET 程序。
    -   **Rider 中 Mono 和 Unity 專案的程式碼覆蓋分析終止：** 由於使用率低和技術債務，Rider 2025.2 將不再為 Mono 和 Unity 專案提供程式碼覆蓋分析。預計在 Unity 遷移到 CoreCLR（預計在 Unity 7 LTS 之後）後，恢復對 Unity 專案的覆蓋分析。
    -   **TeamCity Visual Studio 擴充功能停用：** 為精簡工具並專注於更具影響力的開發者體驗而停止支援。
-   **2025.3 版本更新重點：**
    -   **Rider 中的動態程式分析 (DPA) 演進：** DPA 將不再作為獨立工具存在，其分析能力將進一步整合到 Monitoring 工具中，與 dotTrace 和 dotMemory 分析器一同提供，需 dotUltimate 授權。
-   **未來展望：** 這些變更旨在釋放開發資源，專注於提供使用者最重視的效能提升、穩定性改進和功能優化，以使開發更高效、愉快。

## 配置 App Connect Enterprise 容器化部署的嵌入式全域快取 💾

**Source:** https://dzone.com/articles/configuring-embedded-global-cache-for-app-connect

-   **新快取引入：** App Connect Enterprise (ACE) 13.0.3.0 版本引入了新的「嵌入式全域快取」（Embedded Global Cache）。
-   **替代舊版：** 此新快取取代了舊版的 WebSphere eXtreme Scale (WXS) 嵌入式快取（現在稱為「嵌入式 WXS 網格」），後者已被棄用，但只要 ACE 仍支援 Java 8，它將繼續運作。
-   **核心功能：** 類似於其前身，嵌入式全域快取允許在相同的整合流程、不同流程之間，甚至跨多個整合伺服器儲存和重用資料，從而消除對資料庫等替代持久性解決方案的需求。

## Smart-Doc：在 Java 專案中生成 gRPC API 文件 📝

**Source:** https://dzone.com/articles/smart-doc-generate-grpc-api-docs-java

-   **gRPC 挑戰：** 在現代 Java 微服務中，gRPC 憑藉其高效的二進位協議和多語言支援簡化了服務間通訊，但隨著專案規模擴大，維護 gRPC API 文件變得日益困難。
-   **Smart-Doc 解決方案：** Smart-doc 被視為在 Java 專案中生成 gRPC API 文件的最佳解決方案。
-   **Smart-Doc 優勢：**
    -   **高速：** Smart-doc 設計為快速掃描程式碼並生成文件，無需額外的執行時依賴。它直接提取 `.proto` 檔案，使用 `protoc` 編譯成 Java 程式碼，然後透過解析 Java 程式碼和註解來生成文件。這個過程比一般的 AI 工具更快。
